# Hangman - Lessons Learned
This project taught me that programming is more about **logic** than syntax.

### What I learned

- Break a big problem into smaller steps.
- Use a `while` loop to keep the game running.
- Use a `for` loop to check every letter in the word.
- Use `set()` to remember correct guesses and prevent duplicate guesses.
- Use a boolean (`correct_guess`) to know if the player should lose a life.
- Rebuild the displayed word every round instead of trying to update only one letter.
- Keep track of the game's state using variables like `lives`, `correct_answer`, and `guessed_letters`.

### Biggest takeaway
The biggest lesson I learned is that before writing code, I should first think about **what information my program needs to remember and how it should change after every action.** Once the logic is clear, writing the code becomes much easier.
